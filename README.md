## 📖 项目简介 (Overview)
本项目是一个专为现代物流与供应链场景设计的综合管理系统 (LMS)。系统围绕货物中转、仓储调度与人员权限管理为核心，打造了一个高内聚、低耦合的信息流转平台，旨在提升企业内部流转效率，降低沟通与调度成本。

## ✨ 核心特性 (Key Features)
* **🔐 精细化 RBAC 角色权限控制 (核心亮点)**：
    * 内置完善的动态用户角色修改与分配窗口。
    * 针对真实物流场景深度定制多角色访问控制（包括但不限于：**仓管员 Warehouse Worker、调度员 Dispatcher、司机 Driver、系统管理员 Admin**）。
    * 不同角色登录后动态渲染 UI 与功能菜单，实现数据视图的严格物理隔离与权限越界防护。
* **🏢 高效仓储与调度流转**：实现货物入库、出库、在途状态的实时追踪与数字化管理。
* **📊 数据可视化仪表盘**：直观展示物流节点状态、人员运力分布与库存余量信息。
* **🛡️ 强鲁棒性架构**：基于 MVC/MVVM 架构模式设计，实现底层业务逻辑与上层 UI 视图的彻底解耦，系统具备极强的二次扩展与维护能力。

## 🛠️ 技术栈 (Tech Stack)
* **前端交互**：C++ / Qt 框架, 高级自定义 QSS 样式表
* **后端逻辑**：面向对象设计 (OOP), 基于角色的权限访问控制策略 (RBAC)
* **数据存储**：MySQL 关系型数据库, 安全数据加密机制

## 💡 开发心得 (Developer Notes)
本项目在开发过程中，重点攻克了多角色状态下的 UI 动态刷新与权限校验难题。通过抽象底层 `User` 与 `Role` 的关联关系表，彻底抛弃了硬编码的权限判断，使得未来新增物流业务角色（如：承运商、质检员）只需在数据库中配置即可零代码生效。
